# Employee Payroll & Workforce Management System

## 1. System Overview

The Employee Payroll & Workforce Management System is a role-based web application designed to automate workforce operations including employee management, attendance tracking, leave processing, payroll generation, payslip generation, reporting, notifications, and system administration.

The system supports three user roles:

* Admin
* HR
* Employee

The application follows a Layered Architecture using React, Spring Boot, MySQL, and JWT Authentication.

# 11. Scheduler Architecture

The system contains scheduled jobs.

## Payroll Reminder Scheduler

Checks whether previous month payroll has been generated.

If not:

* Dashboard Reminder
* Email Reminder

## Attendance Reminder Scheduler

Checks missing checkout records after office hours.

Triggers:

* Employee Notification
* Employee Email
* HR Notification

## Notification Cleanup Scheduler

Removes expired notifications after retention period.

---

# 12. Payroll Architecture

Payroll follows snapshot storage.

Payroll stores:

* Employee Name
* Employee Code
* Department
* Designation
* Salary Information

This prevents historical payroll records from changing when employee information changes later.

# 15. Exception Handling Architecture

Global exception handling is implemented.

Handled Exceptions:

* Authentication Errors
* Validation Errors
* Resource Not Found
* Business Rule Violations
* System Exceptions

The application never exposes raw server errors to users.

---

# 16. Soft Delete Strategy

The system does not physically delete:

* Employees
* HR Accounts

Records are marked as:

* ACTIVE
* INACTIVE

Historical data remains available.
